## Цель

Переработать экран `/history` в полноценную «Орбиту дня» — космическую карту дня ребёнка по ТЗ Baby Moon, с интерактивной дугой, карточкой выбранного события и сгруппированной хронологией.

## Новый компонент `src/components/DayOrbit.tsx`

Большая полудуга 6:00 → следующий 6:00 (24ч), стиль «край Земли из космоса».

Слои SVG (снизу вверх):
1. Радиальный фон фиолет-индиго + звёзды (как в `DayArc`).
2. Толстая «атмосферная» дуга — широкий blur-stroke с градиентом лавандовый → синий → бирюза.
3. Тонкая чёткая дуга поверх (граница планеты).
4. Сегменты длительных событий (`sleep`, `walk` с `endedAt`) — цветные дуги поверх трека, цвет по `--sleep`/`--walk`, с `feGaussianBlur` glow.
5. Точки моментальных событий (кормление, подгузник, начало/конец сна без длительности) — светящиеся круги с цветом категории; при клике становятся «выбранным» событием.
6. Текущий момент — маленькое солнце (radial gradient жёлтый) + вертикальная пунктирная линия от солнца к базовой линии под дугой + подпись «Сейчас HH:MM». Только для «Сегодня».
7. Подписи времени под дугой: 6:00, 9:00, 12:00, 15:00, 18:00, 21:00 (равномерно по углу).

Props:
```ts
{ activities: Activity[]; date: Date; now?: number;
  selectedId?: string; onSelect?: (id: string) => void; }
```

Кликабельные `<g>` для каждой точки/сегмента → вызывают `onSelect`. У выбранного — увеличенный радиус и более яркий glow ring.

## Карточка выбранного события `SelectedEventCard`

Внутри `history.tsx`. Стеклянная карточка (`soft-card` + `glow-soft` цвета категории через inline `boxShadow color-mix(--{type})`):
- Эмодзи + название категории (крупно).
- Время: `HH:MM` или `HH:MM — HH:MM`.
- Длительность (для событий с `endedAt`).
- Родитель.
- Доп. поле из `meta` (мл, заметка) если есть.

По умолчанию выбрано последнее событие дня; состояние `useState<string | undefined>`.

## Хронология событий (группировка по частям дня)

Заменить плоский список текущей `history.tsx` на группы:
- **Утро** — 05:00–11:59
- **День** — 12:00–16:59
- **Вечер** — 17:00–22:59
- **Ночь** — 23:00–04:59 (показывать только если есть события)

Внутри группы — список строк:
```
06:42   🌙  Сон закончился        4ч 34м
```
Колонки: время (моно, 56px) · иконка-чип (32px, цвет категории, `glow-soft`) · название · хвост (длительность / объём / родитель, моно, серый). Каждая строка — кликабельная, синхронизирует `selectedId` с дугой.

Группировка по дням оставляем в виде секций сверху, но для текущей версии — один день на экране (выбранный через переключатель).

## Переключатель даты

Маленький блок над сводкой: `←` `Сегодня, 9 мая` `→`. Состояние `selectedDate: Date`. `←/→` сдвигают на ±1 день; правая стрелка disabled, если выбран сегодня. Активити фильтруются по дню.

## Сводка дня

Одна строка под переключателем, моно-разделители ·:
`🌙 {formatDuration(sleep)}  ·  🍼 {feeds}  ·  👶 {diapers}  ·  🚶 {formatDuration(walk)}`.

## Цвета и токены

Используем существующие `--sleep`, `--feed`, `--diaper`, `--walk`. Для «золота подгузника» уже есть `oklch(0.85 0.14 80)` — оставляем. «Жёлтое солнце» — `--accent`. Дополнительных токенов не вводим; для слабого/сильного glow используем существующие утилиты `glow-soft` / `glow-medium` / `glow-strong` и `color-mix(in oklab, var(--{type}) X%, transparent)` в inline стилях.

## UX-детали

- Клик по точке/сегменту дуги или по строке списка — обновляет выбранное событие (двусторонняя синхронизация).
- Если событий нет в выбранный день — показать пустой стейт стеклянной карточкой «В этот день записей нет».
- Длительности форматируются `formatDuration`, время — `formatTime` (24ч, ru-RU) — функции уже есть в `tracker-store`.
- Адаптивно под iPhone: контейнер `max-w-[420px]`, отступы `px-5`, дуга на всю ширину контейнера (`viewBox` сохраняем, `<svg>` `w-full`).
